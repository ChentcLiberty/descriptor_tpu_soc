# 答辩与面试口径

## 一、什么是算子

算子就是一个有明确数学语义的计算单元。

例如：

- 矩阵乘
- 偏置加
- 激活函数
- 卷积
- 梯度更新

你这个项目里已经有算子，不是“没有算子”，只是当前算子被固定调度链路串成了一个小型 MLP 计算流程。

## 二、什么是通用 TPU 编译器

通用 TPU 编译器指的是：

- 输入任意模型或高层程序
- 自动完成：
  - 算子识别
  - 图划分
  - 内存分配
  - tiling
  - 调度
  - 指令生成
  - 数据搬运
- 最后生成加速器可执行命令流

这是一整条大软件栈，不是当前两到三周的目标。

当前更准确的说法是：

> 当前完成的是面向固定小网络的 runtime/driver 和硬件控制链路，不是通用 TPU 编译器。后续会在 descriptor、DMA 和更通用算子抽象之上，逐步向更通用的编译调度栈演进。

## 三、NVIDIA GPU 为什么买来就能用

NVIDIA 卖的不只是硬件，还卖整套软件栈：

- CUDA 编译器
- 驱动
- runtime
- 高性能算子库
- TensorRT 等推理部署工具

所以用户不是“把算法直接塞进电路”，而是：

- 写高层程序
- 经过编译器、库和 runtime
- 最后落到 GPU 上执行

这说明工业界是：

**通用硬件 + 编译器 + runtime + 算子库一起做。**

## 四、你现在做的东西是不是过时

不是。

`寄存器 / descriptor / DMA / runtime / compiler IR / interconnect` 这些不是过时，而是今天大部分加速器系统仍然在用的基础设施。

今天更“新”的方向，不是把这些扔掉，而是在其上做更强的软件栈和更强的 IR/编译优化。

当前行业常见方向包括：

- `torch.compile`
- Triton
- StableHLO / MLIR
- IREE
- TensorRT

但无论这些框架怎么发展，控制面、数据面、任务表达、内存搬运和执行后端依然存在。

## 五、为什么不是所有任务都应该丢给 TPU

因为 CPU 和 TPU 适合做的事情不同。

CPU 适合：

- 控制流
- 预处理
- 特征提取中的不规则逻辑
- 系统管理
- 外设控制

TPU 适合：

- 矩阵乘
- 卷积
- 激活
- update
- 规则、重复、并行的神经网络计算

这就是异构系统的意义。

## 六、为什么当前先做 AXI-Lite，不一步到位做 DMA

推荐答法：

> 当前阶段我先用 AXI-Lite 把控制面做通，因为比赛窗口里最重要的是把 CPU+TPU 的系统闭环跑起来。AXI-Lite 足够承载控制寄存器、模式位和状态回读。后续如果引入 descriptor 和大块数据搬运，我会在此基础上增加 AXI master/DMA，把控制面和数据面分离。

## 七、为什么不删 frontend

推荐答法：

> 当前 frontend 里最有价值的是 IMEM、sequencer、wait 语义和 control_unit 这部分控制内核。它不应该再作为最终 CPU 软件直接操作的接口，但也不应该删除。正确做法是把它降成加速器内部控制层，在外面再包一层更高层的 App Wrapper，让 CPU 只看到任务级接口。

## 八、AXI Arbiter 怎么讲成项目加分点

推荐答法：

> 当前第一阶段系统里主要是 CPU 作为 master、TPU 作为 AXI-Lite 外设，所以仲裁不是瓶颈。我之前做过 AXI arbiter，这为后续扩展到 DMA 场景打了基础。下一阶段如果 TPU 变成 AXI master 去搬运输入、权重和输出，那么 CPU 与 TPU DMA 会共享内存总线，仲裁器就会成为系统级关键模块。

## 九、什么是嵌入式

嵌入式不是单指单片机，而是：

**围绕特定功能做出来的软硬件一体系统。**

你的项目就是典型嵌入式智能系统：

- 有 CPU
- 有外设/总线
- 有加速器
- 有固件
- 有具体应用任务：呼吸识别

## 十、当前项目怎么讲最稳

推荐一句话版本：

> 我现在做的是一个 CPU+TPU 异构呼吸识别系统。CPU 负责预处理和控制，TPU 负责小规模 MLP 推理，通过 AXI-Lite 集成成 SoC 原型。当前阶段我先把推理闭环做通，并预留 infer/train、descriptor、DMA 和更一般网络支持的扩展接口。

## 十一、后续计划怎么讲

推荐分层表达：

### 当前完成

- CPU+TPU 系统联调
- AXI-Lite 控制面
- 固定 MLP inference
- 呼吸识别 demo

### 近期扩展

- `infer/train`
- descriptor
- 多种 MLP 尺寸

### 中期扩展

- DMA
- CPU 与 TPU DMA 多 master 共享内存
- AXI arbiter / interconnect

### 长期扩展

- custom instruction
- 1D CNN
- 更通用的 runtime/编译调度支持
