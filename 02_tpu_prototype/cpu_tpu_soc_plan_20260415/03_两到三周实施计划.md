# 两到三周实施计划

## 一、范围锁定

两到三周内不要试图同时完成：

- 通用 TPU 编译器
- custom instruction
- descriptor
- DMA
- 多种 CNN/MLP 训练全覆盖

正确策略是：

- 先完成一个边界完整的异构 SoC 闭环
- 再把更大的内容定义成后续阶段

## 二、第一阶段必须完成的边界

### 必做

- CPU 核可运行 C 程序
- TPU 可作为 SoC 外设被访问
- `AXI-Lite` 或等价 MMIO 控制路径打通
- 固定 MLP inference 跑通
- 完成呼吸识别 demo

### 强烈建议做

- `MODE=infer/train` 模式位
- `NET_ID` 预留
- `STATUS.busy/done/error`

### 能做就加

- 小规模训练闭环
- 一个简单 debug 页
- 运行周期统计

## 三、分周计划

### 第 1 周：CPU+TPU 真正连起来

目标：

- 明确 CPU 总线接口
- 确认 `AXI-Lite` 访问路径
- 新增 `tpu_app_wrapper`
- CPU 能写 `IN0/IN1/CTRL`
- TPU 能返回 `STATUS/OUT`

交付：

- `CPU -> TPU` 基本通路
- C 驱动最小版
- 最小 MLP inference 测试

如果 AXI-Lite 集成遇阻：

- 退而求其次先做等价 MMIO
- 但接口命名和时序尽量贴近 AXI-Lite，后续好迁移

### 第 2 周：做成呼吸识别 demo

目标：

- 明确呼吸识别输入形式
- CPU 做预处理和特征提取
- 先压成 2 个特征，适配现有 `2x2 MLP`
- TPU 完成分类
- CPU 打印或显示结果

交付：

- `signal -> feature -> TPU -> result` 闭环
- 一套演示脚本或测试流程
- 一页项目流程图

如果真实采样来不及：

- 用 ROM/UART 送固定测试样本
- 答辩时明确“当前演示输入来源”和“真实传感器接入是后续扩展”

### 第 3 周：做一个加分扩展

推荐优先级：

1. `MODE=infer/train` 框架
2. 小规模训练支持
3. descriptor 原型
4. custom instruction 原型

建议顺序：

- 先把 `infer/train` 写进架构
- 再决定是做训练，还是做 descriptor，二选一

如果时间非常紧：

- 只把接口和状态位留出来
- 在答辩材料里清楚说明后续升级路径

## 四、阶段递进路线

这个项目可以很大，但必须分阶段完成。

### 阶段 1：当前比赛可交付

- CPU + TPU + AXI-Lite
- 固定 MLP inference
- 呼吸识别 demo

### 阶段 2：近期扩展

- `infer/train`
- descriptor
- 多种 MLP 尺寸

### 阶段 3：中期扩展

- DMA
- 共享内存
- CPU + TPU DMA 多 master 访存
- AXI arbiter/interconnect

### 阶段 4：长期扩展

- custom instruction
- 1D CNN
- 更一般网络支持
- runtime/编译工具链增强

## 五、为什么这条优先级不割裂

因为每一阶段都复用上一阶段资产：

- 第一阶段的 wrapper 和 frontend 继续复用
- 第二阶段只是在其上增加 descriptor 和训练模式
- 第三阶段只是在系统级增加 DMA 和仲裁
- 第四阶段只是在软件与处理器侧增加更强的启动方式和网络支持

这不是推倒重来，而是层层递进。

## 六、建议锁死的“当前不做”项

为了保证项目能收口，当前明确不做或不强求：

- 任意 C `for` 循环自动映射到 TPU
- PyTorch/ONNX 全自动导入
- 任意网络训练全支持
- 一步到位的 DMA/DDR 系统
- 一步到位的 custom instruction + descriptor + DMA 全栈闭环

## 七、阶段 1 的成功判据

满足下面几条，就算阶段 1 成功：

- CPU 能通过总线访问 TPU
- 固件能调用 TPU 完成一次 inference
- 数据路径和状态路径都可观测
- demo 能输出稳定的呼吸识别结果
- 文档里能清楚讲 current scope 和 future roadmap

## 八、面试/答辩时的项目描述模板

推荐口径：

> 当前我完成了一个 CPU+TPU 异构 SoC 的第一阶段闭环：CPU 负责预处理和控制，TPU 负责小规模 MLP 推理，二者通过 AXI-Lite 集成，并完成了呼吸识别 demo。后续我会在这套架构上继续补 descriptor、训练、DMA 和更通用网络支持。
