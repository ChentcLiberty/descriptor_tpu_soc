## 30 秒版本

“我给自己的定位不是天赋型选手，更偏工程型实习生。基础题我会先讲结论，再补原理和适用边界；项目上我比较重视把简历里的每一句话落到代码、验证结果和 debug 过程。我最近做得最完整的项目，是把一个 tiny-tpu RTL 原型补成 AXI-Lite SoC 形态，并把 forward、backward、参数更新和多 epoch 训练闭环跑通。我不会把没做过的部分讲得很满，但对自己做过的链路，我能讲清楚怎么设计、怎么验证、怎么收敛问题。”

## 1 分钟版本

“我比较偏工程型实习生，不是那种把每个点都讲得特别满的人。我平时会先把基础打扎实，然后把项目做成能落地、能验证、能被追问的状态。最近我主要做的是一个 tiny-tpu SoC 化项目：在原有 RTL 核的基础上，补了 `src_axi/tpu_frontend_axil.sv` 和 `src_axi/tpu_soc.sv` 这条 AXI-Lite 控制路径，用 `compiler/scheduler.py` 和 `compiler/out/imem.hex` 做当前 2x2 原型的指令化执行，再用 cocotb 做系统级验证。这个项目里我在 2026-04-02 复跑了两个关键用例，一个是 `make test_tpu_soc_axil_e2e`，41 个检查点全部通过；另一个是 `make test_tpu_soc_axil_train_convergence`，12 个 epoch 后 loss 从 0.2529 降到 0.1777，最终 XOR 预测是 `(0,1,1,0)`。我比较有把握的部分是系统集成、验证闭环和 debug 收敛；如果超出当前原型边界，比如更大规模映射或者通用编译器，我会明确说这是下一步方向，不会说成已经做完。”

## 90 秒版本

“我给自己的定位是工程型实习生。基础上，我会尽量把数字电路、时序、验证这些题目讲清原理，不会只背结论；项目上，我更关注能不能把系统做通，而不是只讲单个亮点。最近我做得最扎实的项目，是把一个原来更偏 testbench 直驱的 tiny-tpu RTL，补成了带 AXI-Lite 前端和 sequencer 的 SoC 原型。具体来说，我补了 AXI 解码、寄存器堆、启动控制和 IMEM 装载链路，对应文件是 `src_axi/tpu_frontend_axil.sv` 和 `src_axi/tpu_soc.sv`；软件侧我用 `compiler/scheduler.py` 生成当前 2x2、Q8.8、2-layer MLP 的 schedule，再编码成 59 条 IMEM 指令。验证上，我不是只看前向输出，而是做了单次 e2e 闭环和多 epoch 训练闭环两层验证：`test/test_tpu_soc_axil_e2e.py` 会检查 H1、dZ2、dZ1 和 UB 里更新后的参数，2026-04-02 复跑结果是 41/41 PASS；`test/test_tpu_soc_axil_train_convergence.py` 会重复触发 12 个 epoch，最终 loss 从 0.2529 降到 0.1777，预测收敛到 XOR 的 `(0,1,1,0)`。这个过程中我也收敛过几类比较典型的问题，比如 host write 无效、VPU pathway 不能保持、sequencer wait 条件写错导致卡死。我的风格是把结论讲清楚，把证据给出来，把边界说清楚。”

## 自我介绍后可顺接的两句

如果面试官要你接项目：

“如果展开讲，我最能代表自己工程能力的是这个 tiny-tpu SoC 项目，因为它既有 RTL 集成，也有验证闭环，还有比较具体的 debug 过程。”

如果面试官要你接基础：

“我在回答基础题时也会保持同样的风格，先给结论，再解释为什么成立，以及它在什么条件下不成立。”

## 你要避免的自我介绍问题

- 不要一上来就说“我很热爱芯片”“我学习能力很强”，太空。
- 不要把每个项目都堆一遍，面试官记不住。
- 不要说“这个项目所有东西都是我做的”。
- 不要在开场就把自己讲成“架构级选手”。

你的目标只是让对方快速形成一个判断：

**这个人基础不飘，项目真实，沟通有结构，放进团队里比较省心。**
