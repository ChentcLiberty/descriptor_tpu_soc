# 手撕代码题与答案

这份文档的目标不是让你背模板，而是让你在白板或共享文档里，能写出“结构正确、解释得清、不会被一追问就垮”的代码。

## 一、手撕题总原则

1. 先说思路，再写代码。
2. 先写接口和状态，再写细节。
3. 组合逻辑先给默认值。
4. 时序逻辑只放寄存器态，不要混入组合量。
5. 写完主动讲边界和可能优化点。

## 二、题目 1：写一个最小 FSM

### 题目

写一个 3 状态 FSM：`IDLE -> RUN -> DONE`。`start` 拉高进入 `RUN`，`done_i` 拉高进入 `DONE`，下一拍自动回 `IDLE`。

### 参考答案

```systemverilog
typedef enum logic [1:0] {
    IDLE = 2'b00,
    RUN  = 2'b01,
    DONE = 2'b10
} state_t;

state_t state, next_state;

always_ff @(posedge clk or posedge rst) begin
    if (rst)
        state <= IDLE;
    else
        state <= next_state;
end

always_comb begin
    next_state = state;
    case (state)
        IDLE: if (start)  next_state = RUN;
        RUN:  if (done_i) next_state = DONE;
        DONE:              next_state = IDLE;
        default:           next_state = IDLE;
    endcase
end
```

### 你要主动解释的点

- 为什么状态寄存器放 `always_ff`。
- 为什么 `next_state` 要默认等于 `state`。
- 为什么 `default` 不能省。

### 面试官可能追问

Q：为什么不用一个 `always_ff` 全写完？

A：可以，但两段式更利于把状态寄存器和组合跳转逻辑分清，减少漏赋值和错误覆盖。

## 三、题目 2：写一个双缓冲权重寄存器

### 题目

实现一个 Weight Stationary 风格的双缓冲权重寄存器：`accept_weight` 时把新权重装进 inactive bank，`switch_weight` 时把 inactive bank 切到 active。

### 参考答案

```systemverilog
module weight_buffer (
    input  logic        clk,
    input  logic        rst,
    input  logic [15:0] weight_in,
    input  logic        accept_weight,
    input  logic        switch_weight,
    output logic [15:0] weight_active
);
    logic [15:0] weight_inactive;

    always_ff @(posedge clk or posedge rst) begin
        if (rst) begin
            weight_active   <= '0;
            weight_inactive <= '0;
        end else begin
            if (accept_weight)
                weight_inactive <= weight_in;
            if (switch_weight)
                weight_active <= weight_inactive;
        end
    end
endmodule
```

### 你要主动解释的点

- 为什么 active 和 inactive 都是寄存器态。
- 为什么先 preload 再 switch。
- 这个结构和 `src/pe.sv:69-79` 的真实实现是对应的。

### 面试官可能追问

Q：如果 `accept_weight` 和 `switch_weight` 同拍怎么办？

A：要根据需求定义优先级。当前写法里 `switch_weight` 读到的是前一拍的 inactive 值，如果想同拍切新值，就要重写时序语义。

## 四、题目 3：写一个最小 AXI-Lite 寄存器 bank

### 题目

不用写完整 AXI-Lite，只写一个支持 1 个 32-bit 控制寄存器的最小 slave 骨架。

### 参考答案

```systemverilog
logic [31:0] ctrl_reg;
logic        bvalid_r;
logic        rvalid_r;
logic [31:0] rdata_r;

assign s_axil_awready = 1'b1;
assign s_axil_wready  = 1'b1;
assign s_axil_bvalid  = bvalid_r;
assign s_axil_bresp   = 2'b00;
assign s_axil_arready = 1'b1;
assign s_axil_rvalid  = rvalid_r;
assign s_axil_rresp   = 2'b00;
assign s_axil_rdata   = rdata_r;

always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
        ctrl_reg <= 32'b0;
        bvalid_r <= 1'b0;
        rvalid_r <= 1'b0;
        rdata_r  <= 32'b0;
    end else begin
        if (s_axil_awvalid && s_axil_wvalid) begin
            if (s_axil_awaddr[7:0] == 8'h00)
                ctrl_reg <= s_axil_wdata;
            bvalid_r <= 1'b1;
        end else if (bvalid_r && s_axil_bready) begin
            bvalid_r <= 1'b0;
        end

        if (s_axil_arvalid) begin
            if (s_axil_araddr[7:0] == 8'h00)
                rdata_r <= ctrl_reg;
            else
                rdata_r <= 32'h0;
            rvalid_r <= 1'b1;
        end else if (rvalid_r && s_axil_rready) begin
            rvalid_r <= 1'b0;
        end
    end
end
```

### 你要主动解释的点

- AXI-Lite 五通道的基本职责。
- 为什么写响应和读响应都要有 valid/ready 结束条件。
- 这是最小骨架，不是生产级 AXI-Lite slave。

### 面试官可能追问

Q：这个写法有什么简化？

A：这里把 `awready/wready/arready` 全部恒 1 了，没有处理 backpressure，也没有支持地址和数据分拍缓存。真实工程里会更完整。

## 五、题目 4：写一个 valid 管线打一拍

### 题目

给一组 `data_in/valid_in` 打一拍输出 `data_out/valid_out`。

### 参考答案

```systemverilog
always_ff @(posedge clk or posedge rst) begin
    if (rst) begin
        data_out  <= '0;
        valid_out <= 1'b0;
    end else begin
        data_out  <= data_in;
        valid_out <= valid_in;
    end
end
```

### 你要主动解释的点

- 吞吐量不变，但 latency +1。
- 这个结构和 `src/pipeline_exp/vpu_ub_pipe_stage.sv:18-30` 一致。
- 适合切断长组合路径，但前提是上下游协议允许多一拍。

### 面试官可能追问

Q：为什么只打一拍就可能改善 Fmax？

A：因为原本一整条组合链被分成两段，单周期逻辑深度下降，所以 setup 压力会减小。

## 六、题目 5：写一个 Q8.8 乘法后截位逻辑

### 题目

两个 16-bit Q8.8 数相乘，输出仍然是 16-bit Q8.8。

### 参考答案

```systemverilog
logic signed [31:0] mult_full;
logic signed [15:0] mult_q8_8;

always_comb begin
    mult_full = $signed(a) * $signed(b);
    mult_q8_8 = mult_full[23:8];
end
```

### 你要主动解释的点

- 乘法后小数位从 8 变成 16，所以要右移 8 位回到 Q8.8。
- 当前写法是截断，不是舍入。
- 如果担心溢出，还可以再补饱和逻辑。

### 面试官可能追问

Q：为什么取 `[23:8]`？

A：因为 16x16 乘法后结果是 32 bit，其中低 8 位是多出来的小数位，需要去掉，保留中间 16 位即可回到 Q8.8。

## 七、题目 6：写一个简单同步 FIFO

### 题目

写一个深度 4 的同步 FIFO，只要求支持单时钟域。

### 参考答案

```systemverilog
module fifo4 (
    input  logic       clk,
    input  logic       rst,
    input  logic       wr_en,
    input  logic       rd_en,
    input  logic [7:0] din,
    output logic [7:0] dout,
    output logic       full,
    output logic       empty
);
    logic [7:0] mem [0:3];
    logic [1:0] wptr, rptr;
    logic [2:0] count;

    assign full  = (count == 4);
    assign empty = (count == 0);

    always_ff @(posedge clk or posedge rst) begin
        if (rst) begin
            wptr  <= 2'b0;
            rptr  <= 2'b0;
            count <= 3'b0;
            dout  <= 8'b0;
        end else begin
            if (wr_en && !full) begin
                mem[wptr] <= din;
                wptr <= wptr + 1'b1;
            end
            if (rd_en && !empty) begin
                dout <= mem[rptr];
                rptr <= rptr + 1'b1;
            end
            case ({wr_en && !full, rd_en && !empty})
                2'b10: count <= count + 1'b1;
                2'b01: count <= count - 1'b1;
                default: count <= count;
            endcase
        end
    end
endmodule
```

### 你要主动解释的点

- `count` 为什么能直接判断 full/empty。
- 读写同时发生时 `count` 不变。
- 这只是同步 FIFO，异步 FIFO 还要处理跨时钟域同步和 Gray code。

## 八、海思想看的不是“你会不会背模板”，而是 3 个东西

1. 你写完后能不能讲清时序语义。
2. 你能不能主动指出这个写法的边界。
3. 你能不能把题目和项目里的真实代码联系起来。

## 九、手撕代码最容易丢分的地方

1. `always_comb` 里默认赋值没给全。
2. 组合逻辑和时序逻辑混着驱动同一信号。
3. 状态机没有 `default`。
4. valid 打一拍了，data 没打一拍。
5. 乘法只会写，不会解释位宽和缩放。
6. AXI-Lite 只记得通道名字，不会解释握手结束条件。
