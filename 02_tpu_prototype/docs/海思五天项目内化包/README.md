# 海思五天项目内化包

这套资料面向 5 天、每天 4 小时的强化准备，目标不是“多背几页”，而是把你的简历、代码、验证结果、debug 过程和边界口径真正打通，做到海思风格面试里能扛住追问。

## 这套资料怎么用

阅读顺序：

1. `01_五天四小时执行计划.md`
2. `02_项目介绍与口语稿.md`
3. `03_项目详解与代码定位.md`
4. `08_简历逐条证据映射速查.md`
5. `04_项目深挖题库.md`
6. `05_基础知识与答案.md`
7. `06_手撕代码题与答案.md`
8. `07_海思风格表达与模拟面试清单.md`
9. `10_算法组合与执行顺序详解.md`
10. `11_算法组合面试口述版.md`

## 每份文档解决什么问题

- `01_五天四小时执行计划.md`
  解决“这 20 小时具体怎么分配”。
- `02_项目介绍与口语稿.md`
  解决“30 秒 / 90 秒 / 3 分钟怎么讲项目”。
- `03_项目详解与代码定位.md`
  解决“每个结论落到哪个文件、哪个模块、哪段代码”。
- `04_项目深挖题库.md`
  解决“项目相关高频追问怎么答”。
- `05_基础知识与答案.md`
  解决“数字前端、验证、定点、AXI、综合基础怎么答”。
- `06_手撕代码题与答案.md`
  解决“手撕代码怎么写、怎么讲、怎么防止写崩”。
- `07_海思风格表达与模拟面试清单.md`
  解决“海思喜欢什么样的学生，怎么说更像工程型实习生”。
- `08_简历逐条证据映射速查.md`
  解决“简历每一条如何对证据、代码、测试和结果”。
- `10_算法组合与执行顺序详解.md`
  解决“训练算法按顺序对应哪些 VPU/GD 组合，每一步的数据流、公式和代码定位是什么”。
- `11_算法组合面试口述版.md`
  解决“怎么把算法组合压成 30 秒、90 秒和追问场景下能直接说出口的话术”。

## 这套资料默认的人设

统一外显人设只有一句：

**基础扎实、项目真实、边界清楚、能把问题收敛到模块和信号级的工程型实习生。**

## 你最终要做到的状态

- 能用 30 秒、90 秒、3 分钟讲同一个项目，不乱。
- 能把简历 bullet 映射到具体代码和测试。
- 能讲出 3 个最有含金量的 bug 故事。
- 能区分“已经验证过的”和“如果继续做我会怎么做的”。
- 能在高压追问下守住边界，不吹过头。
