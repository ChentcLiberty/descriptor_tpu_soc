# 五天四小时执行计划

总原则：每天只做 4 小时，但每 1 小时都有明确产出。不要一口气乱看文档；要按“讲清楚 -> 对代码 -> 过题 -> 自测”循环推进。

## Day 1：把项目主线讲顺

### 第 1 小时：建立总脑图

读：

- `docs/海思五天项目内化包/02_项目介绍与口语稿.md`
- `docs/PROJECT_INTRO.md`
- `career/interview_prep/01_项目总览与电梯回答.md`

任务：

- 背熟 30 秒版本。
- 用自己的话复述 90 秒版本。
- 在纸上画出 5 块结构：`src/`、`src_axi/`、`compiler/`、`test/`、`docs/`。

产出：

- 能脱稿说出项目目标、三块核心贡献、两个验证结果。

### 第 2 小时：把代码目录和模块对上

读：

- `docs/海思五天项目内化包/03_项目详解与代码定位.md`
- `career/interview_prep/02_代码地图与模块定位.md`

任务：

- 重点记住 `src_axi/tpu_frontend_axil.sv`、`src_axi/tpu_soc.sv`、`src/unified_buffer_v3.sv`、`compiler/scheduler.py`、`test/test_tpu_soc_axil_e2e.py`。
- 对照代码至少打开一次关键文件。

产出：

- 能回答“这个项目最核心的 3 个文件是哪几个，为什么”。

### 第 3 小时：把简历 bullet 讲实

读：

- `docs/海思五天项目内化包/08_简历逐条证据映射速查.md`
- `career/interview_prep/06_简历逐条深问深答.md`

任务：

- 每个 bullet 都写下“代码 + 测试 + 结果 + 边界”。
- 把“我做了什么”和“我没做什么”分开。

产出：

- 能把简历 5 条项目 bullet 各讲 30 秒。

### 第 4 小时：第一次口头模拟

任务：

- 录音 3 次：30 秒、90 秒、3 分钟版本。
- 检查有没有这些问题：
  - 讲太散。
  - 把 scheduler 说成编译器。
  - 把 prototype 说成产品级 SoC。
  - 把 update 说成完整数值收敛验证。

产出：

- 一版能听的项目介绍录音。

## 五天里必须反复提醒自己的 4 句话

1. 先结论，再证据，再边界。
2. 不会的点可以收口，不要防御。
3. 讲系统集成、验证闭环、bug 收敛，不要乱吹“大而全架构”。
4. 海思想听到的是工程判断，不是堆术语。
