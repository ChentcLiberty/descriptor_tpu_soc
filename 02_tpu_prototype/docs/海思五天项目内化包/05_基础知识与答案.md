# 基础知识与答案

这份文档面向数字 IC 设计 / 验证 / 前端综合岗位的高频基础题。答题目标不是写教科书，而是用工程语言讲清原理、适用场景和项目映射。

## 一、时序与 RTL 基础

### Q1：阻塞赋值和非阻塞赋值有什么区别？

A：阻塞赋值按语句顺序立即更新，常用于组合逻辑；非阻塞赋值在当前时刻结束后统一更新，常用于时序逻辑。更重要的是不要混驱同一个信号，否则容易引入竞态和仿真/综合不一致。

### Q2：什么情况下会推导出 latch？

A：组合逻辑块里没有对输出在所有分支做完整赋值时，综合工具为了“记住上一个值”就会推导 latch。最常见的防法是 `always_comb` 里先给默认值，再写分支覆盖。

### Q3：`always_ff` 和 `always_comb` 为什么不能混着驱动同一信号？

A：因为这会形成多源驱动，语义不清，仿真和综合都可能出问题。你项目里 `pe.sv`、`gradient_descent.sv`、`unified_buffer` 的修复，本质上都和“把时序态和组合量分清楚”有关。

### Q4：setup 和 hold 的本质区别是什么？

A：setup 是数据到得太晚，采样边沿到来前还没稳定；hold 是数据到得太早，边沿后很快就改变，破坏了采样窗口。当前项目里你主要在修 setup/Fmax，而不是 hold。

### Q5：什么是关键路径？

A：关键路径是决定当前时钟周期下最差时序裕量的那条路径。找关键路径不是看代码“感觉哪里长”，而是看 timing report 确认跨哪些单元、模块和逻辑级数。

### Q6：什么是多周期路径，什么时候能用？

A：当设计语义允许某条路径不必一个周期到达时，可以声明 multi-cycle path。前提是这条路径在功能上真的是多周期，不是为了让报告好看硬放约束。

## 二、状态机与控制逻辑

### Q7：三段式 FSM 和两段式 FSM 有什么区别？

A：三段式把状态寄存器、next state 和输出逻辑分开，结构更清晰；两段式通常把状态跳转和输出放在一个组合块里。面试里不必站队，关键是默认赋值完整、状态转移清楚、非法状态有兜底。

### Q8：为什么状态机要有 default？

A：因为非法状态或 X 状态总可能出现，default 是最基本的恢复和防御手段。没有 default 的 FSM 在仿真和综合角落情况下都更脆弱。

### Q9：什么是 one-hot 编码，什么时候适合？

A：one-hot 用一个 bit 对应一个状态，组合译码简单、速度快，适合状态不多但追时序的场景；代价是触发器更多。二进制编码更省面积，但组合译码通常更复杂。

## 三、valid/ready 与数据通路

### Q10：valid/ready 协议最核心的规则是什么？

A：只有 `valid && ready` 同时为 1 才算一次真正传输。源端在 `ready=0` 时要保持数据和 valid 稳定；接收端可以通过 ready 施加反压。

### Q11：为什么 ready 到 valid 的组合反馈危险？

A：容易形成长组合环甚至组合环路，导致时序难收敛、仿真难分析。复杂路径上通常要用寄存、skid buffer 或者明确切断组合反馈。

### Q12：什么是 skid buffer？

A：skid buffer 是为了解决下游突然撤掉 ready 时，上游已经发出的一拍数据不能丢的问题。它本质是“1 拍缓冲 + 协议保护”，常用在高频 valid/ready 流水线上。

## 四、定点数基础

### Q13：Q8.8 是什么意思？

A：总位宽 16 bit，其中 8 bit 小数位。它的优点是实现简单、范围和精度都比较平衡，适合这个 tiny-tpu 原型的验证。

### Q14：定点乘法为什么位宽会变宽？

A：两个 16 bit 数相乘会得到更宽的结果，因为整数位和小数位都会叠加。比如 Q8.8 乘 Q8.8 后，小数位变成 16 bit，因此要做截位、缩放或舍入才能回到目标格式。

### Q15：截断、舍入、饱和分别是什么？

A：截断是直接丢低位，简单但误差偏大；舍入会根据被丢掉部分决定是否进位，数值更好；饱和是在超出表示范围时钳住最大/最小值，避免回绕。面试里至少要说明你知道这些 trade-off。

### Q16：为什么多轮训练里 fixed-point 很难逐 bit 对齐？

A：因为每轮都会发生截断、缩放和更新次序差异，小误差会随 epoch 累积。所以多 epoch 更适合看行为收敛，而不是每一步逐 bit 一样。

## 五、AXI-Lite 基础

### Q17：AXI-Lite 和 AXI4 的主要区别是什么？

A：AXI-Lite 只做单次寄存器式读写，没有 burst，适合控制面；AXI4 支持高吞吐、突发传输，更适合数据面。

### Q18：AXI-Lite 有哪五个通道？

A：写地址 AW、写数据 W、写响应 B、读地址 AR、读数据 R。写和读是彼此独立的，写地址和写数据也可以不同拍到达。

### Q19：为什么最小 AXI-Lite slave 也不能只盯一个 always 块瞎写？

A：因为 AW/W/B 和 AR/R 是两套握手机制。最小实现也要清楚处理地址、数据、响应的时序关系，否则会出现重复采样或漏响应。

### Q20：你的项目里 AXI-Lite 实际承担了什么？

A：承担寄存器配置、UB 数据写入、IMEM 地址/数据写入、start/step 控制和状态回读。它是控制面，不是高吞吐数据搬运接口。

## 六、存储与 FIFO 基础

### Q21：同步 FIFO 最少要讲清哪些点？

A：写指针、读指针、full/empty 判断、读写同时发生时的行为、复位后状态。面试里如果手写 FIFO，先保证语义清楚，再谈性能优化。

### Q22：寄存器堆和 SRAM 的工程差别是什么？

A：寄存器堆灵活、容易综合、便于小规模原型；SRAM 面积更省，但需要宏、接口和更完整的工艺支持。你这个项目里 UB 更像寄存器化存储，所以面积热点也明显。

## 七、综合与 PPA 基础

### Q23：为什么用 `compile_ultra`？

A：因为它更适合做时序驱动的激进优化，现代 DC flow 中很常见。对这个项目来说，核心诉求是追关键路径和 Fmax 收敛，所以它比基础 `compile` 更合适。

### Q24：retiming 是什么？

A：retiming 是工具自动移动寄存器位置以改善时序。它有用，但如果你需要一个可解释、可控、可 A/B 对照的结构级改动，显式 pipeline 变体往往更稳。

### Q25：为什么只堆大 drive strength 不一定能解决时序？

A：如果根因是长组合逻辑深度或跨模块长路径，单纯换大门只能局部改善，收益有限，还可能显著增加面积和功耗。结构性切断路径通常更有效。

### Q26：为什么要强调 DRC 0 违例？

A：因为综合不只是看频率，还要看设计规则是否满足。真正可用的结果应该是在约束和 DRC 合法前提下达到目标频点。

## 八、验证基础

### Q27：reference model 的核心作用是什么？

A：给硬件结果一个独立可信的参考，不让你只盯波形猜功能对不对。reference model 越接近硬件数值语义，验证结论就越稳。

### Q28：scoreboard 的作用是什么？

A：把“观察结果”变成“自动判定”，减少人眼看波形的主观性。项目里 `test/test_tpu_soc_axil_e2e.py` 的 scoreboard 就是把 H1、dZ2、dZ1 和 UB 参数检查自动化。

### Q29：为什么 directed test 仍然重要？

A：因为复杂系统最后一个顽固 bug 往往需要可控、可复现、可逐拍观察的场景。随机或大系统回归能发现问题，但未必能高效定位根因。

### Q30：覆盖率的价值是什么？

A：覆盖率不是“证明没 bug”，而是帮助你发现哪些代码、状态和条件根本没被打到。它是验证完整性的辅助证据，不是功能正确性的唯一证据。

## 九、项目映射型基础题

### Q31：为什么你项目里 `vpu_pathway` 要 latch，这和基础知识有什么关系？

A：本质是控制和数据时序对齐问题。它要求你理解脉冲信号、流水线延迟和时序保持，而不只是看组合功能是否正确。

### Q32：为什么 `wait_after` 要靠 bit 编码，这和基础知识有什么关系？

A：本质是状态机控制语义要显式、可验证。把等待语义编码成数据而不是写死在控制逻辑里，可维护性和可扩展性都会更好。

### Q33：为什么 pipeline 变体比直接硬改原 top 更像工程做法？

A：因为它保留基线，方便 A/B 对照、验证复用和问题回退。这体现的是工程管理和实验设计意识，不只是会写 RTL。

## 十、答题方法提醒

回答基础题时尽量用这个模板：

1. 先说定义。
2. 再说为什么重要。
3. 最后落回项目里的一个例子。

示例：

“setup violation 就是数据到得太晚，边沿来时还没稳定。它重要是因为直接限制 Fmax。这个项目里我追到的关键路径就是 VPU 到 UB 的跨模块 setup 路径，所以最后用了显式 pipeline 变体去切断它。”
