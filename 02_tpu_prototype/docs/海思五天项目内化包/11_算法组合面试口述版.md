## 先记一句总纲

“当前这个 2-layer MLP 训练流里，真正有意义的组合按顺序就是五类：`1100 + off` 做第一层 forward，`1111 + bias(B2)` 做第二层 transition，`0001 + bias(B1)` 做 hidden backward，最后两步 `0000 + weight GD` 分别更新 `W1` 和 `W2`。所以这个项目不是随便拼控制位，而是每一种组合都对应一段明确的训练语义。”

## 30 秒版本

“这个训练流我会按五步讲。第一步是 `1100 + off`，先算 `X @ W1^T`，再做 `bias + leaky_relu`，得到 `H1`。第二步是 `1111 + bias(B2)`，先算 `H1 @ W2^T`，再串起输出梯度和激活导数，直接得到 `dZ2`，同时更新 `B2`。第三步是 `0001 + bias(B1)`，先算 `dH1 = dZ2 @ W2`，再结合 UB 里的 `H1` 得到 `dZ1`，同时更新 `B1`。最后两步都是 `0000 + weight GD`，分别通过外积更新 `W1` 和 `W2`。”

## 90 秒版本

“我会把这条训练链拆成五个有意义的组合。第一阶段是 hidden layer forward，systolic 先算 `X @ W1^T`，VPU 走 `1100`，也就是 `bias -> leaky_relu`，输出 `H1` 并写回 UB。第二阶段是 output transition，systolic 先算 `H1 @ W2^T`，VPU 走 `1111`，也就是 `bias、激活、输出梯度、激活导数` 这整条链，最后直接得到 `dZ2`；因为 `db2` 本质上就是这条 `dZ2` 流，所以 `B2` 可以在 UB 内并行更新。第三阶段是 hidden backward，systolic 先算 `dH1 = dZ2 @ W2`，VPU 只开 `0001` 的导数路径，把 `dH1` 和 UB 里保存的 `H1` 结合成 `dZ1`，同时更新 `B1`。第四和第五阶段都不需要 VPU，所以设成 `0000`，分别通过外积得到 `dW1` 和 `dW2`，再由 UB 里的 gradient_descent 模块原地更新 `W1` 和 `W2`。整个设计的关键不是某个单点算子，而是每一步的数据流、控制流和参数更新都能对上算法语义。”

## 每一步的最短说法

### 1. `1100 + off`

“第一层 forward。systolic 算 `X @ W1^T`，VPU 做 `bias + leaky_relu`，输出 `H1`。”

### 2. `1111 + bias(B2)`

“第二层 transition。systolic 算 `H1 @ W2^T`，VPU 串起输出梯度链，直接产出 `dZ2`，并顺手更新 `B2`。”

### 3. `0001 + bias(B1)`

“hidden backward。systolic 先算 `dH1 = dZ2 @ W2`，VPU 只做导数，把 `dH1` 和 `H1` 结合成 `dZ1`，并更新 `B1`。”

### 4. `0000 + weight(W1)`

“`W1` 更新。VPU 不参与，systolic 直接做 `dW1 = dZ1^T @ X` 的外积，然后 GD 更新 `W1`。”

### 5. `0000 + weight(W2)`

“`W2` 更新。VPU 不参与，systolic 直接做 `dW2 = dZ2^T @ H1` 的外积，然后 GD 更新 `W2`。”

## 面试官最容易追问的 6 个点

### 1. 为什么不是任意排列，而是固定顺序？

“因为 VPU 的硬件级联顺序在 RTL 里写死了，永远是 `bias -> leaky_relu -> loss_derivative -> leaky_relu_derivative`。`vpu_data_pathway` 只是在这个固定顺序上决定每一级开还是关，不是任意换顺序。”

### 2. 为什么 `1111` 要缓存 `last_H`？

“因为当前流里刚算出的 `H2` 过了 `loss_derivative` 之后，主数据流就变成 `dL/dH2` 了，但最后一级导数还需要原始 `H2`，所以必须单独缓存一份。”

### 3. 为什么 `0001` 用 UB 的 `H_in`？

“因为这一步当前流里传过来的已经不是激活，而是 `dH1`。对应的 `H1` 是前面 forward 已经写进 UB 的，所以导数模块要从 UB 取 `H_in`。”

### 4. 为什么 `0001` 对应 `dZ1`？

“因为 hidden backward 的公式就是 `dZ1 = dH1 * f'(H1)`。systolic 先算 `dH1 = dZ2 @ W2`，VPU 这里只需要做导数这一段，所以路径自然是 `0001`。”

### 5. bias 更新为什么不需要单独矩阵乘？

“因为 bias 梯度不是外积。`db2` 本质上来自 `dZ2`，`db1` 本质上来自 `dZ1`，所以直接把 delta 流送进 gradient_descent 就够了；只有 weight gradient 才需要外积路径。”

### 6. 为什么 weight update 是 `0000`？

“因为 weight update 只需要外积和参数更新，不需要 bias、激活、loss derivative 或 activation derivative，所以 VPU 设成全旁路，纯靠 systolic + GD 完成。”

## 最后压成 5 句背诵版

1. “`1100 + off`：第一层 forward，得到 `H1`。”
2. “`1111 + bias(B2)`：第二层 transition，得到 `dZ2`，同时更新 `B2`。”
3. “`0001 + bias(B1)`：hidden backward，得到 `dZ1`，同时更新 `B1`。”
4. “`0000 + weight(W1)`：外积更新 `W1`。”
