# 算法组合与执行顺序详解

这份文档只解决一个问题：

**当前这个 2-layer MLP + 2x2 + Q8.8 + training 原型里，训练算法按顺序到底对应哪些有意义的 VPU/GD 组合，每一步的数据怎么流，为什么是这样。**

## 先定边界

这里不要把三个层次混在一起：

1. **VPU 控制空间**
   `vpu_data_pathway` 是 4 bit，所以理论上有 `2^4 = 16` 种 mask。
2. **VPU 运算顺序**
   顺序不是任意排列，RTL 里永远固定为：
   `bias -> leaky_relu -> loss_derivative -> leaky_relu_derivative`
3. **当前算法真正使用的组合**
   对当前 `2-layer MLP` 训练流，真正有明确算法语义、被 scheduler 使用并验证过的核心组合只有 5 类。

## 一张总表

| 顺序 | 算法阶段 | Systolic 在做什么 | VPU mask | GD 状态 | 结果 |
|---|---|---|---|---|---|
| 0 | host preload | 无 | 无 | 无 | X/Y/W/B 先装入 UB |
| 1 | forward layer1 | `X @ W1^T` | `1100` | off | 得到 `H1` |
| 2 | transition layer2 | `H1 @ W2^T` | `1111` | bias(B2) | 得到 `dZ2`，同时更新 `B2` |
| 3 | backward layer1 | `dZ2 @ W2` | `0001` | bias(B1) | 得到 `dZ1`，同时更新 `B1` |
| 4 | update W1 | `dZ1^T @ X` | `0000` | weight(W1) | 得到 `dW1` 并更新 `W1` |
| 5 | update W2 | `dZ2^T @ H1` | `0000` | weight(W2) | 得到 `dW2` 并更新 `W2` |

## 先记 UB 的读选择

后面每一步都要用到 `ub_ptr_select`，先记住这几类：

- `ptr_sel=0`：给 systolic/VPU 流输入数据
- `ptr_sel=1`：给 PE 装权重
- `ptr_sel=2`：给 VPU 的 bias
- `ptr_sel=3`：给 VPU 的 `Y`
- `ptr_sel=4`：给 VPU 的 `H_in`
- `ptr_sel=5`：给 GD 读取旧 bias 值
- `ptr_sel=6`：给 GD 读取旧 weight 值

## Stage 0: Host preload

这一步没有复杂组合，就是 host 通过 AXI 把 `X / Y / W1 / B1 / W2 / B2` 写入 UB。

这一步的意义是：

- 后续 sequencer 不再靠 testbench 逐拍推内部控制
- 后续所有训练阶段都从 UB 按事务读出数据

面试时可以这样讲：

“训练开始前，host 先把输入、标签和参数全部写入 UB，后续执行由 IMEM 指令和 UB 事务读控制，不再靠 testbench 手工逐拍驱动。”

## Stage 1: forward layer1

公式：

`Z1 = X @ W1^T + B1`

`H1 = leaky_relu(Z1)`

对应 scheduler：

- [scheduler.py](/home/jjt/tpu-soc/compiler/scheduler.py#L124)

这一阶段的组合是：

`Systolic 矩阵乘 + VPU 1100 + GD off`

具体过程：

1. 用 `ptr_sel=1` 把 `W1^T` preload 到 PE 的 shadow weight 路径
2. 经过若干 `nop` 后执行 `switch`
3. 用 `ptr_sel=0` 流入 `X`
4. 同时用 `ptr_sel=2` 流入 `B1`
5. VPU 使用 `1100`，即 `bias -> leaky_relu`
6. 最终输出 `H1`，并写回 UB

为什么是 `1100`：

- bit3=1：开 bias
- bit2=1：开 leaky relu
- bit1=0：不开 loss derivative
- bit0=0：不开 activation derivative

这一阶段不更新参数，因为当前只是前向生成中间激活，还没有梯度。

面试时可以这样讲：

“第一阶段是 hidden layer forward。systolic 先算 `X @ W1^T`，VPU 用 `1100` 补上 `bias + leaky_relu`，最后把 `H1` 写回 UB，供后续第二层 forward 和 hidden backward 复用。”

## Stage 2: transition layer2

公式：

`Z2 = H1 @ W2^T + B2`

`H2 = leaky_relu(Z2)`

`dL/dH2 = loss_derivative(H2, Y)`

`dZ2 = dL/dH2 * f'(H2)`

对应 scheduler：

- [scheduler.py](/home/jjt/tpu-soc/compiler/scheduler.py#L179)

这一阶段的组合是：

`Systolic 矩阵乘 + VPU 1111 + GD bias(B2)`

具体过程：

1. 先 preload `W2^T`
2. 用 `ptr_sel=0` 流入 `H1`
3. 用 `ptr_sel=2` 流入 `B2`
4. 用 `ptr_sel=3` 流入 `Y`
5. 用 `ptr_sel=5` 读取旧 `B2`，给 bias update 路径准备旧值
6. VPU 使用 `1111`
7. 主输出为 `dZ2`
8. 同时在 UB 内把 `B2` 原地更新

为什么 `1111` 合理：

- bit3=1：`bias`
- bit2=1：`leaky_relu`
- bit1=1：`loss_derivative`
- bit0=1：`leaky_relu_derivative`

这一步的关键理解点有两个。

### 1. `loss_parent` 不是算标量 loss，而是算输出梯度

代码自己就写了这个 TODO：

- [vpu.sv](/home/jjt/tpu-soc/src/vpu.sv#L150)

所以这里更准确的说法是：

“这一级把 `H2` 和 `Y` 结合，生成 `dL/dH2`，而不是单独计算一个标量 loss。”

### 2. 为什么这里要缓存 `last_H`

因为 `leaky_relu` 输出的是 `H2`，但经过 `loss` 之后，主数据流就已经变成 `dL/dH2` 了。后面的 `lr_d` 还需要原始 `H2` 去算 `f'(H2)`，所以需要把 `H2` 单独缓存下来。

- [vpu.sv](/home/jjt/tpu-soc/src/vpu.sv#L332)

这一步还能顺手更新 `B2`，原因是：

- `db2` 本质上就是输出层 delta 流，也就是 `dZ2`
- UB 内部的 `gradient_descent` 模块直接把写回流作为 `grad_in`

面试时可以这样讲：

“第二阶段不是单纯第二层 forward，而是 output transition。systolic 先算 `H1 @ W2^T`，VPU 用 `1111` 串起 `bias、激活、输出梯度和激活导数`，直接产出 `dZ2`。同时因为 `db2` 就是 `dZ2` 这条流，所以 UB 内可以并行更新 `B2`。”

## Stage 3: backward layer1

公式：

`dH1 = dZ2 @ W2`

`dZ1 = dH1 * f'(H1)`

对应 scheduler：

- [scheduler.py](/home/jjt/tpu-soc/compiler/scheduler.py#L261)

这一阶段的组合是：

`Systolic 矩阵乘 + VPU 0001 + GD bias(B1)`

具体过程：

1. 先 preload `W2`
2. 注意 backward 这里使用 `W2`，不转置
3. 用 `ptr_sel=5` 先装旧 `B1`，把 bias-update 路径提前“武装好”
4. 用 `ptr_sel=0` 流入 `dZ2`，systolic 先算 `dH1`
5. 用 `ptr_sel=4` 再流入 `H1`，给 `lr_d` 提供激活信息
6. VPU 使用 `0001`
7. 主输出为 `dZ1`
8. 同时在 UB 内把 `B1` 原地更新

为什么 `0001` 合理：

- 这里不需要 bias
- 不需要再做一次正向 `leaky_relu`
- 不需要 loss derivative
- 只需要 `activation derivative`

也就是说这一阶段本质上就是：

`dZ1 = dH1 * f'(H1)`

为什么这一步不用 `last_H`，而要用 UB 的 `H_in`：

- 当前这次流里传过来的已经不是新激活，而是 `dH1`
- 对应的 `H1` 是前面第一阶段 forward 已经算好并写进 UB 的
- 所以这一步 `lr_d` 应该从 UB 取 `H_in`

相关代码：

- [vpu.sv](/home/jjt/tpu-soc/src/vpu.sv#L299)
- [scheduler.py](/home/jjt/tpu-soc/compiler/scheduler.py#L293)
- [scheduler.py](/home/jjt/tpu-soc/compiler/scheduler.py#L307)

这一步也可以顺手更新 `B1`，因为：

- `db1` 本质上就是 `dZ1`
- 所以 `dZ1` 这条输出流可以直接喂给 GD 模块

面试时可以这样讲：

“第三阶段是 hidden layer backward。systolic 先算 `dH1 = dZ2 @ W2`，VPU 用 `0001` 只开导数路径，把 `dH1` 和 UB 里保存的 `H1` 结合成 `dZ1`。因为 `db1` 就是 `dZ1`，所以 `B1` 也在这一阶段原地更新。”

## Stage 4: update W1

公式：

`dW1 = dZ1^T @ X`

`W1 <- W1 - lr * dW1`

对应 scheduler：

- [scheduler.py](/home/jjt/tpu-soc/compiler/scheduler.py#L330)

这一阶段的组合是：

`Systolic 外积 + VPU 0000 + GD weight(W1)`

具体过程：

1. 用 `X` 作为一侧输入
2. 用 `dZ1^T` 作为另一侧输入
3. systolic 形成外积，得到 `dW1`
4. 用 `ptr_sel=6` 读取旧 `W1`
5. `gradient_descent` 模块完成 `W1 <- W1 - lr * dW1`

为什么这里 VPU 是 `0000`：

- weight update 不需要 bias
- 不需要 activation
- 不需要 loss derivative
- 不需要 activation derivative

所以这里 `0000` 不是“空转”，而是“VPU 不参与，这一步纯靠 systolic + GD 完成”。

相关代码：

- [scheduler.py](/home/jjt/tpu-soc/compiler/scheduler.py#L352)
- [scheduler.py](/home/jjt/tpu-soc/compiler/scheduler.py#L366)

面试时可以这样讲：

“第四阶段是 `W1` 的外积更新。systolic 形成 `dW1 = dZ1^T @ X`，VPU 设成 `0000` 纯旁路，不做额外算子，UB 里的 `gradient_descent` 模块读取旧 `W1` 并完成原地更新。”

## Stage 5: update W2

公式：

`dW2 = dZ2^T @ H1`

`W2 <- W2 - lr * dW2`

对应 scheduler：

- [scheduler.py](/home/jjt/tpu-soc/compiler/scheduler.py#L382)

这一阶段的组合是：

`Systolic 外积 + VPU 0000 + GD weight(W2)`

具体过程：

1. 用 `H1` 作为一侧输入
2. 用 `dZ2^T` 作为另一侧输入
3. systolic 形成外积，得到 `dW2`
4. 用 `ptr_sel=6` 读取旧 `W2`
5. `gradient_descent` 模块完成 `W2 <- W2 - lr * dW2`

这一步和 `W1` 更新是同构的，区别只是外积两边的数据换成了 `dZ2^T` 和 `H1`。

面试时可以这样讲：

“第五阶段是 `W2` 更新，和 `W1` 更新同构，只是外积两边换成 `dZ2^T` 和 `H1`。这一步同样是 `0000 + weight GD` 的纯更新路径。”

## 为什么 bias 更新不需要单独一条矩阵乘路径

这是一个很容易被追问的点。

权重梯度需要外积：

- `dW1 = dZ1^T @ X`
- `dW2 = dZ2^T @ H1`

所以 weight update 一定要经过 systolic 路径。

但 bias 梯度不是外积：

- `db1` 本质上来自 `dZ1`
- `db2` 本质上来自 `dZ2`

所以 bias update 不需要再单独算一次矩阵乘，而是直接把 `dZ` 流作为 `grad_in` 送给 `gradient_descent` 模块。

## 为什么 `1111` 用 `last_H`，`0001` 用 UB 的 `H_in`

这是 VPU 里最容易被问晕的点，直接记下面这句：

**当前流里新生成的激活，用 `last_H`；前向阶段早就存下来的激活，用 UB。**

具体来说：

- `1111` 时，当前这次 VPU 流里刚刚产生 `H2`，但过完 `loss_derivative` 后主流变成 `dL/dH2`，所以必须缓存 `H2`
- `0001` 时，当前流里传过来的是 `dH1`，对应的 `H1` 不是当前新产生的，而是前面 forward 已经写进 UB 的，所以从 `H_in` 取

## 你最后应该能说出的总版本

“当前这个 2-layer MLP 训练流里，真正有意义的组合按顺序就是五类。第一步 `1100 + off` 做第一层 forward，得到 `H1`；第二步 `1111 + bias(B2)` 做第二层 transition，直接产出 `dZ2` 并更新 `B2`；第三步 `0001 + bias(B1)` 做 hidden backward，产出 `dZ1` 并更新 `B1`；第四和第五步都是 `0000 + weight GD`，分别用外积更新 `W1` 和 `W2`。所以这个项目不是随便拼控制位，而是每一种组合都对应一段明确的训练语义。”

## 最后再压成 5 句记忆版

1. `1100 + off`：`X @ W1^T` 后做 `bias + leaky_relu`，得到 `H1`
2. `1111 + bias(B2)`：`H1 @ W2^T` 后做输出梯度链，得到 `dZ2`，同时更新 `B2`
3. `0001 + bias(B1)`：`dZ2 @ W2` 后只做导数，得到 `dZ1`，同时更新 `B1`
4. `0000 + weight(W1)`：外积得到 `dW1`，更新 `W1`
5. `0000 + weight(W2)`：外积得到 `dW2`，更新 `W2`
